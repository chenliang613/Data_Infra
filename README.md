# 数据基础设施，分五层：网络层、算力层、流通层、利用层、安全层

## 数据空间 — 通过Agent，实现点对点文件共享演示

本地同时运行 5 个 Agent 节点：1 个全局节点 + 4 个 P2P 节点，每个节点提供 Web 管理界面，支持文件共享、互信握手、文件获取与数据市场。

---

## 快速启动

### 1. 安装依赖

```bash
pip install -e .
```

### 2. 启动所有节点

```bash
python demo/start_p2p.py
```

启动后自动打开 5 个浏览器标签：

| 节点 | 地址 | 角色 |
|------|------|------|
| 全局节点 Agent-ALL | http://localhost:8020 | 全局监控 |
| 数据共享方 Agent-A | http://localhost:8021 | 共享方 |
| 数据共享方 Agent-B | http://localhost:8023 | 共享方 |
| 数据接收方 Agent-C | http://localhost:8025 | 接收方 |
| 数据接收方 Agent-D | http://localhost:8027 | 接收方 |

> 启动时会自动终止占用以上端口的进程。

#### 启动选项

```bash
python demo/start_p2p.py           # 检测到历史数据时交互询问
python demo/start_p2p.py --load    # 直接加载历史数据恢复上次状态
python demo/start_p2p.py --fresh   # 忽略历史数据，全新启动
```

### 3. 单独启动某个节点（可选）

```bash
# 单个 P2P 节点
python demo/p2p_agent.py --port 8021 --name "我的节点"

# 全局节点
python demo/global_agent.py --port 8020
```

## 功能

### 四个Agent节点互相共享文件（8021 / 8023 / 8025 / 8027）

#### 共享文件
- 点击或拖拽上传文件（支持任意类型、可多选）
- 对已共享文件执行预览、下载、删除
- 点击「📢 发布」将共享文件发布到数据市场，填写发布说明后确认；已发布的文件可随时「撤回」

#### 发布数据需求
- 在「共享文件」Tab 底部填写需求标题与描述，发布至全局节点的数据市场需求信息列表
- 已发布的需求支持随时撤回

#### 已接收文件
- 查看从其他节点获取的所有文件
- 对已接收文件执行预览、下载、删除
- 重名文件自动追加时间戳后缀（如 `data1_20260301_153012.csv`）

#### 互信管理
- 输入对方节点地址，选择互信等级后发起请求，可附带说明信息
- 接受或拒绝传入的互信请求
- 对已互信节点浏览其共享文件并一键获取

#### 互信等级

| 等级 | 说明 |
|------|------|
| 🔐 高互信 | 可浏览文件列表 + 下载文件 |
| 👁 一般互信 | 仅可浏览文件列表，不能下载 |

---

### 全局节点 Agent-ALL（8020）

实时汇总所有节点的共享、互信与访问、数据市场等情况，提供以下视图：

| Tab | 内容 |
|-----|------|
| 📊 节点概览 | 各节点在线状态、统计数据、最近 3 个共享文件与已接收文件（可跳转至对应节点对应模块） |
| 📤 全部共享文件 | 汇总所有节点共享文件列表，含节点标签、大小、上传时间 |
| 📥 全部已接收文件 | 汇总所有节点已接收文件列表，含来源节点、接收时间 |
| 🔐 互信关系 | 全网互信关系汇总 |
| 📋 访问日志 | 全网文件访问记录（预览、下载等事件） |
| 🛒 数据市场 | 汇总所有节点发布的供给信息与需求信息 |

#### 清除全部历史

在 Agent-ALL 页面右上角提供「🗑 清除全部历史」按钮，点击并确认后将同步清除所有节点的：

- 共享文件（包括实际文件）
- 已接收文件
- 互信关系
- 市场发布与需求
- 访问日志

清除完成后页面自动刷新，恢复至初始空白状态。下次启动 demo 时同样不会读取旧历史。

## 状态持久化

每个 P2P 节点在数据目录下维护一个 `state.json`，节点运行期间所有变更（上传文件、建立互信、发布市场信息等）都会实时写入。下次启动时自动恢复：

- 共享文件元数据（实际文件仍在 `shared/` 目录）
- 已接收文件元数据（实际文件仍在 `received/` 目录）
- 互信关系与等级
- 市场供给与需求发布
- 访问日志

> 若文件已被手动删除，加载时会自动跳过对应记录，不会出现悬空引用。

## 演示流程

### 共享方和接收方 文件共享

```
1. 在共享方 Agent-A「共享文件」Tab 上传文件
2. 在「互信管理」Tab 输入接收方地址，选择互信等级，点击「发起互信」
   例：http://localhost:8025
3. 切换到接收方 Agent-C，「互信管理」Tab 出现角标 → 点击「接受」
4. 双方状态变为绿色「已互信」
5. 在接收方「互信管理」点击对方的「浏览文件」→ 选中文件 → 点击「获取」
6. 「已接收文件」Tab 出现该文件，可预览 / 下载
7. 在 Agent-ALL（8020）「节点概览」或「访问日志」查看全局动态
```

### 数据市场

```
1. 在任意节点「共享文件」Tab，点击某文件的「📢 发布」→ 填写发布说明 → 确认
2. 在任意节点「共享文件」Tab 底部，填写数据需求标题与描述 → 点击「发布需求」
3. 在 Agent-ALL（8020）「🛒 数据市场」Tab 查看全网供给与需求汇总
```

### 重置演示

```
1. 打开 Agent-ALL（8020）
2. 点击右上角「🗑 清除全部历史」→ 在弹窗中确认
3. 所有节点数据清空，恢复初始状态，可重新开始演示
```

|------|--------|
| 图片 | jpg · jpeg · png · gif · webp · svg · bmp · ico · avif |
| PDF | pdf |
| 视频 | mp4 · webm · mov · avi · mkv · m4v |

## 架构

```
                  全局节点 Agent-ALL (:8020)
                  ┌──────────────────────┐
                  │  节点概览             │
                  │  全部共享文件         │
                  │  全部已接收文件       │
                  │  互信关系             │
                  │  访问日志             │
                  │  数据市场             │
                  │  [清除全部历史]       │
                  └──────────┬───────────┘
                             │ 轮询各节点 API
        ┌────────────────────┼────────────────────┐
        ▼                    ▼                    ▼                    ▼
Agent-A (:8021)      Agent-B (:8023)      Agent-C (:8025)      Agent-D (:8027)
┌─────────────┐      ┌─────────────┐      ┌─────────────┐      ┌─────────────┐
│  共享文件    │      │  共享文件    │      │  共享文件    │      │  共享文件    │
│  (发布市场)  │      │  (发布市场)  │      │  (发布市场)  │      │  (发布市场)  │
│  (发布需求)  │      │  (发布需求)  │      │  (发布需求)  │      │  (发布需求)  │
│  已接收文件  │      │  已接收文件  │      │  已接收文件  │      │  已接收文件  │
│  互信管理    │      │  互信管理    │      │  互信管理    │      │  互信管理    │
└──────┬──────┘      └──────┬──────┘      └──────┬──────┘      └──────┬──────┘
       └──────────────────── P2P ──────────────────────────────────────┘
```

每个节点数据独立存储：

```
data/p2p/
├── agent-8021/
│   ├── shared/      # 该节点正在共享的文件
│   ├── received/    # 该节点已接收的文件
│   └── state.json   # 持久化状态（文件元数据、互信关系、市场数据等）
├── agent-8023/
├── agent-8025/
└── agent-8027/
```
